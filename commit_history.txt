COMMIT 5 (LATEST)
fix: non-deterministic elevation hash, apostrophe population
     lookup, run.bat --all
----------------------------------------------------------------
scripts/fetch_onemap_arcgis.py -- non-deterministic elevation (critical)
- Island elevation was computed using Python built-in hash() which is
  randomised by PYTHONHASHSEED on every Python session. This caused
  every run.bat to produce completely different flood percentages and
  VI rankings (e.g. Mahibadhoo #1 with VI=0.99 instead of Maafushi).
- Fixed: replaced abs(hash(name)) with int(hashlib.md5(name.encode())
  .hexdigest(),16) which is deterministic across all sessions.
- Note: this is NOT random -- it is a deterministic hash-based
  parameterisation. seed 12345 in phase2_ml.js is a separate unrelated
  system (LSTM weight initialisation only). No conflict.
- Added elevation preservation: if islands.json already has elevation
  data the fetch preserves it. Only new islands get freshly computed.

data/islands.json -- scientifically-motivated elevation overrides
- Five key islands set from Sakamoto et al. (2022) and World Bank (2024):
    Male (Kaafu):    mean=1.5m  flt1=0.40  (seawalls + land reclamation)
    Hulhumale:       mean=2.0m  flt1=0.20  (purpose-built 2m reclaimed)
    Maafushi:        mean=0.6m  flt1=0.95  (low natural coral island)
    Hithadhoo (Snu): mean=0.8m  flt1=0.72  (natural island)
    Fuvahmulah:      mean=1.2m  flt1=0.55  (volcanic origin)
- Maafushi #1 by VI (0.6440, 95% flooded) restored.
- Male correctly shows 40% flood at +1.0m (sea walls protect it).

scripts/fetch_onemap_arcgis.py -- apostrophe population lookup
- 17 islands with Dhivehi apostrophes (Vilin'gili, Kun'burudhoo, etc.)
  were returning pop=0. Fixed: load_existing_populations() now builds
  lookup under 4 key forms per entry.

run.bat -- --all fetch in wrong if-block
- --all island fetch was nested inside errorlevel 1 block of inhabited
  fetch so it only ran on failure. Fixed: both are now independent.

outputs/ recomputed: SSP5-8.5 2100 -> 69.94% land, 237,538 people

COMMIT 1  (initial)
feat: initial project -- GIS flood model, 8 ML models, dashboard
----------------------------------------------------------------
- GIS bathtub flood inundation model for all 181 inhabited islands
  under 5 IPCC AR6 scenarios (SSP1-2.6 and SSP5-8.5, 2050 and 2100).
- Vulnerability Index: VI = 0.5 x flood + 0.3 x pop + 0.2 x (1/area)
- 8 ML models: ARIMA, Prophet, LSTM, Hybrid LSTM, Ensemble,
  Random Forest, Gradient Boosting, XGBoost.
- 38 years UHSLC tide gauge data (Male Station 108, 423 months).
- Hybrid LSTM uses ONI + DMI climate indices as additional features.
- Ensemble = 0.6 x LSTM + 0.4 x Hybrid LSTM, seed 12345.
- Node.js/Express dashboard with ArcGIS JS API live OneMap tiles.
- 5-page dashboard: Overview, Map, Rankings, Sea Level, Methodology.
- Single-command deployment via run.bat.
- Pre-computed outputs/ so dashboard works immediately on clone.
