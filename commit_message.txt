fix: inhabited filter, population data, Ensemble crash, all-islands
     toggle, glossary

Inhabited island filter (scripts/fetch_onemap_arcgis.py)
- is_inhabited() was defined but never called in the main loop, causing
  all 1132 OneMap features to pass through instead of 181 inhabited
  islands only. Added call after the name check.
- The category field whitelist was then too strict: most inhabited islands
  have a blank or unexpected category value in OneMap, so only 22 atoll
  capital islands passed through. Rewrote is_inhabited() to use Census
  2022 exact name match as the primary signal. An island is kept if its
  normalised name (apostrophes and accents stripped) matches any census
  entry with population > 0.
- Removed partial/substring matching which caused uninhabited islands with
  similar names (Vilin'gilivarufinolhu, Vilin'gilivaru, En'boodhoo) to
  incorrectly pass, giving 277 islands instead of 181.
- Removed plain 'island' from INHABITED_CATEGORIES (too broad). The
  two-word value 'inhabited island' is retained as a fallback.
- Added --all flag via argparse. When passed, skips the is_inhabited()
  filter and writes all features to data/islands_all.json. Inhabited-only
  run behaviour is unchanged.

Population data (data/islands.json, scripts/fetch_onemap_arcgis.py)
- Population lookup now uses 'name (atoll)' as the primary key to handle
  the 9 island names that exist in multiple atolls (Meedhoo, Dhidhdhoo,
  Goidhoo, Feydhoo, Magoodhoo, Guraidhoo, Omadhoo, Thinadhoo, Hithadhoo).
  Plain name fallback retained for unique names.
- Corrected population values for 14 affected entries in islands.json
  using Census 2022 atoll-qualified figures. Total population corrected
  to 492,119.

Ensemble future projection crash (scripts/phase2_ml.js)
- Ensemble future projection loop threw TypeError when NaN propagated
  through LSTM weights during the ~900-step rollout to 2100. Added
  isFinite guard and try/catch with last-valid-value fallback to prevent
  crash at line 687.

All-islands toggle (public/index.html, public/js/dashboard.js,
                    scripts/phase1_gis.js, scripts/run_all.js, run.bat)
- scripts/phase1_gis.js: runPhase1() accepts optional suffix parameter.
  All saveJSON calls append the suffix so inhabited and all-islands
  outputs do not overwrite each other.
- scripts/run_all.js: after the main GIS run, checks for islands_all.json
  and if present runs a second GIS pass writing _all suffix outputs.
- run.bat: added second fetch_onemap_arcgis.py --all call after the
  inhabited fetch. Failure of the --all call is non-fatal.
- public/js/dashboard.js: state.allScenariosAll added. boot() silently
  tries to fetch all_scenarios_all.json after main data load. Failure is
  silent. renderRankings() switches to the extended dataset when the
  toggle is checked and data is available. Shows a setup instruction
  message if data has not yet been generated.
- public/index.html: checkbox added to Rankings page with two conditional
  messages: one noting that uninhabited island VI reflects land exposure
  only, one pointing to the setup command if data is unavailable.
